Todolists Service 层说明

本目录是业务服务层：在 tRPC 与数据库 Repository 之间，负责入参校验、权限边界（userId）、调用仓储、以及出参映射。 对外契约由 Zod Schema 定义；实现类通过依赖注入获取 TodoRepository。

03

各文件实现逻辑

todolists-service-impl.ts — 核心流程

每个公开方法遵循相同模式（list 略简，无 data 解构）：

1. parseRequest(RequestSchema, request)   // 入参 Zod 校验
2. 使用 ctx.userId（不信任客户端传的 userId）
3. todoRepository.*(...)                // 持久化
4. todolists*ResponseMapper(...)        // 出参映射 + parseResponse

update / delete 在写库前先 assertTodoOwnedByUser：按 id 查行，若不存在或 user_id !== ctx.userId 则抛统一「未找到」错误，避免信息泄露。

todolists-request.schema.ts

约定请求体统一包在 { data: ... } 中（list 除外为空对象）。

  • create：todo 字符串 + completed（创建时 impl 强制 false）
  • update：含 id（UUID）、todo、completed
  • delete：仅 id

todolists-response.schema.ts

对外响应同样 { data: ... } 包装。

  • list：数组项仅 id, todo, completed（隐藏 user_id）
  • create：含完整行含 user_id、时间戳
  • update：无 user_id，有时间戳
  • delete：仅返回被删 id

todolists-mapper.ts

Repository 类型为 TodoCreateResponse（表行）。mapListRow 裁剪字段；各 mapper 调用 parseResponse(Schema, payload)，校验失败抛 INVALID_RESPONSE。

todolists-service.ts

接口方法均接收 (request, ctx: { userId: string })。userId 由 tRPC protectedProcedure 从上下文注入，Service 永不从 request body 读取用户身份。

DI 注册： library/services/registrations.ts 将 TodolistsServiceImpl 包装在 serviceErrorProxy 中，并注入 MainDbTokens.todoRepository。

新增功能推荐编写顺序

以本项目的分层为准：自内向外先稳定数据与契约，再暴露 API。下表以「新增 toggleComplete 或批量删除」类需求为例。

1

数据库 / Repository（若需要新查询）

在 library/db/main/todolists/ 扩展 TodoRepository 接口与 TodoRepositoryImpl，必要时改 Drizzle schema。Service 只依赖接口，不直接写 SQL。

2

todolists-request.schema.ts

新增 *RequestSchema 与 z.input 类型；保持 { data: ... } 包装约定（与现有 create/update 一致）。

3

todolists-response.schema.ts

定义返回结构；思考 list 与 detail 是否暴露不同字段（参考 list 隐藏 user_id 的做法）。

4

todolists-service.ts

在接口上声明新方法签名，带上 ctx: { userId: string }。

5

todolists-mapper.ts

新增 todolists*ResponseMapper，Repository 行 → 对外 DTO，末尾 parseResponse。

6

todolists-service-impl.ts

实现方法：parseRequest → 权限/归属检查（可复用 assertTodoOwnedByUser）→ Repository → Mapper。写操作务必用 ctx.userId 过滤或校验。

7

tRPC Router + 前端

library/trpc/routers/todolists/todolists-router.ts 增加 procedure；页面调用 trpc.todolists.*。无需改 registrations.ts，除非新建了另一个 Service 类。

国际化与 SEO（可选）

config/locale/*/todolists.ts、页面文案等。
